# Self-Diagnosis 유저플로우

> 멤버가 `/자가진단` 슬래시 커맨드로 음성 활동 건강도를 진단한다. 봇 커맨드(`apps/bot/src/command/voice-analytics/`), API 진단 서비스(`apps/api/src/voice-analytics/self-diagnosis/`), 웹 정책 설정·진단 페이지(`apps/web/app/dashboard/guild/[guildId]/diagnosis/`)가 연동된다. 뱃지 시스템과 주간 리포트 설정을 포함한다.
> 🔒 마커: 멤버 음성 활동·교류 데이터(PII) 집계, AI 분석 결과 처리 줄.

---

## UF-SD-001: 자가진단 실행 (봇 슬래시 커맨드)

### 입력
- 멤버가 `/자가진단` 슬래시 커맨드를 실행

### 처리
1. (봇 공통 전제: 길드 컨텍스트 확인 — DM이면 안내 후 종료)
2. 응답을 Ephemeral로 지연 처리(deferred)
3. API에 진단 요청 (서버·사용자 식별자 전달)
4. API: 기능 활성화 여부 확인 → 비활성화이면 거부 사유 반환
5. 쿨다운 활성화 시 쿨다운 키 확인 → 진행 중이면 남은 시간과 함께 거부
6. 🔒 분석 기간 내 멤버의 활동량·활동일·순위·백분위, 교류 인원·관계 다양성(편중도)·주요 교류 상대, 모코코 기여 점수·순위, 마이크 사용률·혼자 비율을 집계 (PII 음성 활동/관계 데이터)
7. 정책 기준 대비 항목별 충족/미달 판정 산출
8. 보유 뱃지 조회 + 미달성 뱃지 달성 가이드 구성
9. 쿨다운 활성화 시 쿨다운 키 설정
10. AI 요약 모드가 켜진 경우 진단 결과를 단기 캐시에 저장
11. 봇: 진단 결과 수신 후 활동 기록이 전혀 없으면 활동 없음 안내 후 종료
12. AI 요약 모드일 때 별도 요청으로 LLM 요약 생성 시도 — 성공 시 요약을, 실패 시 상세 데이터 섹션으로 폴백
13. 결과를 Ephemeral Embed로 렌더링 (활동량/관계 다양성/모코코/참여 패턴/뱃지 + 분석 기간·다음 진단 가능 시각 푸터)

### 출력
- 성공: 본인에게만 보이는 진단 Embed
- 거부/오류: 본인에게만 보이는 사유 안내

### 엣지케이스
- DM에서 실행 → 서버 전용 안내
- 기능 비활성화 → 비활성화 안내
- 쿨다운 진행 중 → 남은 시간 안내 후 거부
- 분석 기간 내 활동 기록 전무 → 활동 없음 안내
- 🔒 관계 다양성(Co-Presence) 데이터 없음 → 관계 다양성 섹션을 데이터 부족으로 표시, 나머지 정상 (PII 부재 처리)
- 모코코 데이터 자체가 서버에 없음 / 본인 모코코 활동 없음 → 각각 별도 안내 문구로 분기 표시
- LLM 요약 생성 실패 → AI 섹션 생략하고 상세 데이터로 폴백
- 🔒 LLM 할당량 초과 → AI 할당량 초과 안내 Embed (AI 처리)
- 쿨다운 비활성화 설정 → 무제한 사용, 푸터에 제한 없음 표기

## UF-SD-002: 진단 정책·뱃지 기준 설정 (웹)

### 입력
- 관리자가 진단/설정 페이지에서 기능 활성화, 분석 기간, 쿨다운 활성화·시간, AI 요약 활성화, 정책 기준(최소 활동 시간·활동일 비율·관계 다양성 점수·최소 교류 인원), 뱃지 기준(활동왕/사교왕/헌터/꾸준러/소통러)을 입력하고 저장

### 처리
1. (공통 전제: 웹 진입 + 서버 관리 권한 검증 통과)
2. 입력 유효성 검사 (각 항목 허용 범위)
3. 관계 다양성·사교왕 점수는 표시용 점수(높을수록 다양)에서 내부 편중도 원본값으로 역변환하여 저장
4. API가 정책 설정을 영속화(upsert)하고 설정 캐시 갱신

### 출력
- 저장 성공: 성공 알림, 이후 진단·뱃지 판정에 즉시 반영
- 저장 실패: 오류 알림

### 엣지케이스
- 허용 범위 밖 입력(분석 기간/쿨다운/백분위 등) → 저장 전 차단
- AI 요약 모드 ON 저장 → 진단 Embed가 상세 데이터 대신 AI 요약 위주로 전환
- 쿨다운 비활성화 저장 → 쿨다운 시간 입력 숨김, 무제한 사용
- 설정 변경 직후 진단 실행 → 캐시 갱신으로 새 기준 적용

## UF-SD-003: 뱃지 배치 계산 및 프로필 카드 표시

### 입력
- 시스템 스케줄러(매일 자정 이후 정해진 시각)에 의한 자동 트리거
- 멤버의 `/me` 프로필 카드 조회

### 처리
1. 스케줄러: 기능 활성화된 모든 서버의 정책 설정 조회
2. 🔒 서버별로 전체 멤버의 활동량 순위·관계 편중도·모코코 순위·마이크 사용률·활동일 비율을 집계 (PII 전체 멤버 음성 데이터)
3. 각 멤버의 뱃지 자격 판정 후 뱃지 스냅샷을 일괄 저장(upsert)
4. `/me` 프로필 카드 조회 시 해당 멤버의 보유 뱃지를 카드 헤더에 pill 형태로 렌더링

### 출력
- 스케줄러: 멤버별 뱃지 스냅샷 갱신
- 프로필 카드: 보유 뱃지 표시(없으면 미표시)

### 엣지케이스
- 자가진단 비활성화 서버 → 배치 대상 제외
- 선행 데이터(관계/모코코) 없는 멤버 → 해당 뱃지 미부여
- 뱃지 4개 초과 보유 → 우선순위 순으로 잘라 표시
- 뱃지 없음 → 프로필 카드 레이아웃 기존과 동일(별도 행 미표시)
- 진단 시 뱃지는 배치 스냅샷 기준이므로 당일 변동분은 다음 배치까지 미반영 가능

## UF-SD-004: 주간 리포트 설정 및 자동 발송

### 입력
- 관리자가 주간 리포트 설정(활성화, 발송 채널, 발송 시각 등) 저장
- 시스템 스케줄러에 의한 주간 자동 트리거

### 처리
1. (공통 전제: 웹 진입 + 서버 관리 권한 검증 통과)
2. 리포트 설정 저장(upsert)
3. 스케줄러가 정해진 주기에 활성화된 서버를 순회
4. 🔒 서버 음성 활동·건강도 지표를 집계하여 요약 리포트 구성 (PII 서버 단위 음성 데이터)
5. 설정된 채널에 요약 Embed 자동 전송

### 출력
- 설정 저장: 성공/실패 알림
- 자동 발송: 지정 채널에 주간 요약 Embed 게시

### 엣지케이스
- 리포트 비활성화 서버 → 발송 대상 제외
- 발송 채널 미설정/삭제 또는 🔒 봇 권한 부족 → 발송 실패, 로그 기록 (권한)
- 집계 대상 데이터 없음 → 데이터 부족 형태의 리포트 또는 발송 생략
- 🔒 AI 요약 포함 시 할당량 초과 → AI 섹션 생략 또는 폴백 (AI 처리)

# Newbie 유저플로우

> 신규 멤버 관리: 환영인사, 미션 추적, 모코코 사냥(동반 플레이 기록), 신입기간 역할 자동관리. 봇 이벤트/인터랙션 핸들러(`apps/bot/src/event/newbie/`, `apps/bot/src/event/guild-member/`), API 로직(`apps/api/src/newbie/`), 웹 설정·관리(`apps/web/app/settings/guild/[guildId]/newbie/`, `apps/web/app/dashboard/guild/[guildId]/newbie/`)가 연동된다.
> 🔒 마커: 멤버 식별자·닉네임(PII) 처리, 역할 부여/제거·강퇴 등 권한 조작, DM 발송 줄.

## UF-NEWBIE-001: 신규 멤버 가입 온보딩 (봇 이벤트)

### 입력
- 새 멤버가 서버에 가입 (가입 이벤트가 트리거)

### 처리
1. 🔒 가입 멤버 정보(식별자·표시 닉네임·아바타·가입 시각 등)를 길드 멤버로 항상 동기화(upsert) — 신입 설정과 무관 (PII 멤버 정보)
2. API에서 신입 관리 설정 조회 — 설정 없으면 종료
3. 환영인사 활성화 + 환영 채널 설정 시: 봇이 템플릿 변수(닉네임/멘션/멤버수/서버명)를 치환하여 환영 Embed를 환영 채널에 직접 전송
4. 미션 활성화 시: API에 미션 생성 요청 (시작일·마감일·목표 산정)
5. 🔒 신입역할 활성화 + 역할 식별자 설정 시: 봇이 Discord에서 멤버에게 신입 역할 부여 후 API에 통보하여 신입기간 레코드 생성 (권한 조작)

### 출력
- 환영 채널에 환영 메시지 게시
- 미션 레코드 생성
- 신입 역할 부여 + 신입기간 시작

### 엣지케이스
- 설정 자체가 없음 → 온보딩 전체 생략 (단, 길드 멤버 동기화는 수행)
- 환영 채널을 찾을 수 없거나 텍스트 채널이 아님 → 환영 전송 생략
- 🔒 환영 전송 시 봇 권한 부족 → 로그 기록 후 조용히 실패 (권한)
- 환영 썸네일 URL 미설정 → 멤버 아바타로 대체
- 미션/역할 각 기능은 독립 활성화 — 일부만 켜진 조합 가능
- 🔒 역할 부여 실패(권한 부족, 역할 미존재) → 로그 기록, 신입기간 레코드 미생성 가능 (권한)
- 길드 멤버 동기화 실패 → 로그 기록 후 온보딩 계속 진행
- 봇 멤버가 가입 → 미션/모코코 대상에서 후속 처리 시 제외

## UF-NEWBIE-002: 미션 진행 추적 및 만료 판정

### 입력
- (자동) 가입 시 생성된 미션
- 멤버의 음성 채널 활동 누적 (voice 도메인 데이터)
- 시스템 스케줄러(매일 자정 KST) 트리거
- 미션 현황 메시지의 갱신 버튼 클릭

### 처리
1. 멤버의 음성 플레이타임을 집계 — 설정에 따라 채널 접속 시간 또는 마이크 ON 시간 기준으로 분기
2. 플레이횟수를 세션 단위로 카운트 — 최소 참여시간/시간 간격 옵션으로 유효 세션 필터링·병합
3. 달성 판정: 플레이타임만 기준(목표 횟수 미설정) 또는 플레이타임 AND 플레이횟수(목표 횟수 설정 시) 모두 충족
4. 스케줄러: 마감일이 지난 진행중 미션을 조회하여 목표 달성 여부 재확인 후 완료/실패로 갱신
5. 영향받은 서버의 미션 캐시 무효화, 미등록 멤버 자동 등록, 미션 현황 메시지 갱신
6. 갱신 버튼 클릭 시(봇): 길드 식별자 파싱 → API에 미션 현황 갱신 요청 → 갱신 완료 안내
7. 미션 현황은 설정된 표시 방식(Embed 또는 Canvas 이미지)으로 알림 채널에 표시

### 출력
- 미션 현황 메시지(Embed 또는 Canvas) 게시/갱신
- 마감 도래 미션의 완료/실패 상태 확정

### 엣지케이스
- 마이크 시간 기준 옵션 변경 → 진행 중 모든 미션의 누적 시간이 새 기준으로 재계산 (관리자에게 사전 경고)
- 활동 기록 없는 멤버 → 진행중 유지, 마감 시 실패 처리
- 🔒 멤버가 마감 전 서버 탈퇴 → 미션을 퇴장 상태로 전환하고 현황 메시지에서 숨김 (PII/멤버 상태)
- 봇 멤버의 미션 레코드 → 갱신 시 삭제
- 갱신 버튼 customId에 길드 식별자 누락 → 잘못된 요청 안내
- 표시 방식 전환(Embed↔Canvas) → 양쪽 설정·템플릿은 독립 보존
- 개별 미션 처리 실패 → 로그 기록 후 나머지 미션 계속 처리

## UF-NEWBIE-003: 모코코 사냥 집계 (동반 플레이 기록)

### 입력
- 기존 멤버와 신규 멤버(모코코)가 같은 음성 채널에 동시 접속 (동시접속 틱·세션 종료 이벤트가 트리거)

### 처리
1. 모코코 기능 활성화 여부 확인 — 비활성화이면 종료
2. 가입 후 기준 일수 이내 멤버를 모코코로 판정 (cutoff 기준)
3. 채널에 모코코가 1명 이상이면 사냥꾼별로 채널 기반 시간 1분 누적 (모코코 수 무관), 모코코별 상세 시간은 참고용 별도 누적
4. 사냥꾼 자격: 설정에 따라 기존 멤버만 또는 모코코 포함 전체 (자기 자신 사냥 제외)
5. 세션 종료 시 유효성 판정 — 세션 시간이 최소 동시접속 시간 이상이면 유효 세션으로 저장 및 횟수 카운트, 미달이면 시간·횟수 롤백 후 무효 세션으로 기록
6. 플레이횟수 카운팅: 최소 참여시간 누적·시간 간격 병합 옵션 적용
7. 점수 재계산(세션 수×가중치 + 채널 시간×가중치 + 고유 모코코 수×가중치) 후 순위 갱신
8. 음성 제외 채널/카테고리의 동시접속은 집계 대상에서 제외
9. 기간별 리셋 설정(누적/월간/커스텀)에 따라 순위 캐시 초기화 (DB 이력은 영구 보존)

### 출력
- 사냥꾼별 점수·순위 갱신
- 모코코 사냥 순위 알림(Embed 또는 Canvas) 게시/갱신

### 엣지케이스
- 채널에 모코코 없음 → 누적 생략
- 사냥꾼 자격 미달(모코코 본인, 옵션 OFF) → 집계 제외
- 세션 시간이 최소 동시접속 시간 미달 → 무효 처리 및 롤백
- 제외 채널/카테고리 → 동시접속이라도 미집계
- 동시접속 시간 간격이 병합 기준 이내 → 동일 1회로 병합 (중복 카운트 방지)
- 누적 상태가 메모리에 보관됨 — 프로세스 재시작 시 진행 중 누적분 초기화 가능
- 리셋 주기 도래 → 순위 캐시만 초기화, 과거 세션·일별 집계 보존

## UF-NEWBIE-004: 모코코 순위 조회 (봇 버튼 상호작용)

### 입력
- 사용자가 모코코 순위 메시지의 이전/다음/갱신/내 순위 버튼을 클릭

### 처리
1. (봇 공통 전제: 버튼 인터랙션 수신, customId 접두사로 분기)
2. 갱신/페이지 이동: 메시지 업데이트를 지연 처리하고 API에서 해당 페이지 랭킹 데이터를 받아 표시 방식(Embed/Canvas)에 맞게 원본 메시지를 수정
3. 내 순위: Ephemeral 응답을 지연 처리하고 본인 상세(점수·시간·세션·도움준 모코코)를 본인에게만 표시

### 출력
- 순위 메시지 갱신/페이지 전환
- 내 순위: 본인에게만 보이는 개인 상세

### 엣지케이스
- 페이지 이동 customId 파싱(길드 식별자·현재 페이지) → 경계 페이지에서 조회 결과에 따라 처리
- 본인 순위 데이터 없음 → 데이터 없음 안내
- API 통신 실패 → 일반 오류 안내 (이미 응답/지연 시 followUp)
- 표시 방식이 Canvas → 이미지 첨부, Embed → Embed 갱신으로 분기

## UF-NEWBIE-005: 신입기간 역할 자동 제거 (스케줄러)

### 입력
- 시스템 스케줄러(매일 자정 KST) 트리거

### 처리
1. 만료일이 지난 활성 신입기간 레코드 조회
2. 서버별 신입 역할 식별자 확인
3. 🔒 봇이 Discord에서 해당 멤버의 신입 역할 제거 (권한 조작)
4. 신입기간 레코드를 만료 처리
5. 영향받은 서버의 관련 캐시 무효화

### 출력
- 신입 역할 제거, 신입기간 만료 확정

### 엣지케이스
- 만료 대상 없음 → 무처리 종료
- 미션 완료 여부와 무관 — 신입기간(일수) 경과만 기준
- 🔒 역할 제거 실패(권한 부족, 역할/멤버 미존재) → 로그 기록 후 다음 레코드 계속 처리. 레코드는 만료 처리 (권한)
- 기간 만료 후 다른 역할로 교체하지 않음(제거만 수행)
- 조회·캐시 무효화 실패 → 로그 기록 후 진행

## UF-NEWBIE-006: 미션 수동 관리 (웹 대시보드)

### 입력
- 관리자가 미션 관리 화면에서 상태 필터로 미션을 조회하고, 진행중 미션을 성공/실패 처리하거나, 미션의 Embed 표시/숨김을 토글

### 처리
1. (공통 전제: 웹 진입 + 서버 관리 권한 검증 통과)
2. 상태 필터·페이지로 미션 목록 조회 (기본 진행중)
3. 성공 처리: 미션을 완료로 갱신, 🔒 (옵션) 선택한 역할을 멤버에게 부여 (권한 조작)
4. 실패 처리: 미션을 실패로 갱신, 🔒 (옵션) 강퇴 — 강퇴 전 (옵션) DM으로 사유 발송 후 멤버 강퇴 (권한 조작 + DM/PII)
5. Embed 토글: 미션의 숨김 여부 전환
6. 처리 후 미션 캐시 무효화 및 현황 메시지 갱신

### 출력
- 상태 변경/역할 부여/강퇴/숨김 반영
- 처리 결과(필요 시 경고 포함) 응답

### 엣지케이스
- 성공/실패 처리는 진행중 미션에만 허용
- 존재하지 않는 미션 식별자 → 대상 없음 응답
- 다른 서버 소속 미션 식별자 → 검증 차단
- 🔒 역할 부여 실패 → 미션 상태는 이미 갱신됨, 경고 메시지 동반 응답 (권한)
- 🔒 강퇴 실패(권한 부족, 멤버 미존재) → 미션 상태는 이미 갱신됨, 경고 메시지 동반 응답 (권한)
- 🔒 DM 차단된 멤버 → DM 발송 조용히 실패하고 강퇴는 계속 진행 (DM/PII)
- 숨김 처리된 미션 → 현황 메시지에서 제외되나 웹에서는 조회 가능

## UF-NEWBIE-007: 신입 관리 설정 (웹)

### 입력
- 관리자가 설정 페이지의 4개 탭(환영인사/미션/모코코 사냥/신입기간)에서 각 기능 활성화 및 세부 설정(채널·템플릿·기간·목표·점수 가중치·역할 등)을 입력하고 저장

### 처리
1. (공통 전제: 웹 진입 + 서버 관리 권한 검증 통과)
2. 탭별 입력 유효성 검사 (템플릿 변수 유효성 포함 — 미지원 변수 사용 시 저장 차단)
3. API가 설정을 영속화(upsert)하고 캐시 갱신
4. 표시 방식·템플릿 변경분은 다음 메시지 갱신 시 반영

### 출력
- 저장 성공: 성공 알림, 이후 온보딩·미션·모코코 동작에 반영
- 저장 실패: 오류 알림

### 엣지케이스
- 미지원 템플릿 변수 사용 → 프론트·백엔드 양쪽에서 저장 차단
- 마이크 시간 기준 변경 → 진행 중 미션 재계산 경고 (UF-NEWBIE-002 연계)
- 각 기능 독립 활성화 — 일부만 켠 상태 저장 가능
- 🔒 신입 역할 식별자 미설정 상태로 역할 기능 활성화 → 가입 시 역할 부여 미동작 (권한 설정 누락)
- 표시 방식 전환 → 반대 방식 설정·템플릿 데이터 보존

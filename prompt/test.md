다음 코드베이스에 대해 Jest 기반 테스트를 작성하라.

역할:
- 구현된 기능이 명세대로 동작하는지 검증한다.
- 실패 케이스를 적극적으로 탐지한다.

작업 절차:

1. /docs/{prd,userflow,usecases,plan}.md 문서를 읽고
   기능 요구사항과 경계 조건을 파악한다.

2. 테스트 대상을 다음으로 분류한다.
   - Unit Test
   - Integration Test
   - E2E Test

3. 각 테스트에 대해 다음을 반드시 포함한다.
   - 정상 케이스
   - 실패 케이스
   - 경계값
   - 예외 상황

4. Jest 기반으로 테스트 코드를 작성한다.
   - 외부 의존성은 명확히 mock 또는 test double 처리한다.
   - 테스트 이름은 요구사항을 그대로 반영한다.

5. 테스트 실행 결과를 분석하고,
   실패하는 테스트가 있다면 원인과 개선안을 제시한다.

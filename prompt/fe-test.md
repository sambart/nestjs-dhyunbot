다음과 같이 프론트엔드 테스트를 진행합니다.

## FE 테스트 원칙 (Testing Trophy)

1. **통합 테스트가 주력이다** — 유저 행동 기반, 페이지 단위
2. **단위 테스트는 순수 로직만** — 유틸 함수, 계산 로직, 복잡한 훅
3. **구현 결합 금지** — 리팩토링만으로 깨지는 테스트는 작성하지 않는다

### 문서 분석 (Specification Analysis)

구현 코드를 보기 전에 **반드시 문서만 먼저 분석**하세요.

#### 읽어야 할 문서
- `/docs/specs/prd/_index.md`, `/docs/specs/prd/{domain}.md`
- `/docs/specs/userflow/_index.md`, `/docs/specs/userflow/{domain}.md`
- `/docs/specs/database/_index.md`, `/docs/specs/common-modules.md`
- `/docs/plans/pages/N-name/{plan,state}.md`
- `/docs/external/*.md`

#### 문서에서 반드시 추출할 것
- 유저 시나리오 ("사용자가 X를 하면 Y가 나타난다")
- 성공/실패/에러 시나리오별 기대 결과
- 에러 코드 및 발생 조건
- 보안 요구사항 (검증, 인증, 인가)
- 제외 사항 / 실패 조건

### 테스트 레벨 가이드

#### 통합 테스트로 검증할 것 (주력)
- 페이지 로드 → 데이터 표시 → 유저 액션 → 결과 확인 (전체 시나리오)
- 폼 입력 → 유효성 검증 → 제출 → 성공/실패 피드백
- API 실패 시 에러 메시지 표시 및 복구 동작
- 권한 없음 / 접근 불가 상태 처리
- 로딩 → 성공 → 에러 상태 전이

#### 단위 테스트로 검증할 것 (선별)
- 순수 유틸 함수 (날짜 포맷, 점수 계산, 데이터 변환)
- 복잡한 비즈니스 로직이 있는 커스텀 훅
- shared 패키지의 유틸리티 함수

#### 작성하지 말 것
- 단순 렌더링 확인 (smoke test)
- props 전달 여부 확인
- DOM 구조 / CSS 클래스 검증
- 내부 state 직접 검증
- 함수 호출 여부만 확인하는 spy 테스트

---

### 테스트 체크리스트 생성 (Checklist Creation)

#### 체크리스트 구성 기준
- "사용자가 ~하면 ~가 보인다/변한다" 형태의 유저 관점 문장
- 성공 / 실패 / 경계 시나리오 포함
- 통합 테스트 시나리오 우선, 순수 로직만 단위 테스트 별도 표기

#### 우선순위
- 🔴 필수: 유저 시나리오 통합 테스트 (폼 제출→결과 확인, 인증 흐름, 데이터 변경)
- 🟡 중요: 에러 핸들링, 권한별 UI, 로딩 상태 (통합 테스트 내에서 함께 검증)
- 🟢 권장: 순수 유틸 함수 단위 테스트, 경계값

#### 스스로에게 물어볼 질문
- 이 테스트가 **리팩토링만으로 깨질 수 있는가?** → Yes면 작성하지 않는다
- **유저 행동 관점에서 검증하는가?** → No면 작성하지 않는다
- 사용자가 잘못 입력했을 때 **침묵하면** 실패인가?
- 서버 실패 시 UI가 정상처럼 보이면 실패인가?

---

### 최종 테스트 보고 (Test Report)

#### 반드시 보고할 내용
1. **테스트 요약**
   - 총 테스트 수
   - Integration / Unit 분포
   - 유저 시나리오 커버리지

2. **검증 결과**
   - 유저 시나리오별 Acceptance Criteria 충족 여부
   - 커버되지 않은 시나리오 목록

3. **발견된 문제**
   - 유저 시나리오 실패
   - 에러 상황 침묵 (메시지 미노출)
   - 잠재적 버그 / 보안 취약점

4. **차단 의견 (Blocking Issues)**
   - 릴리즈 불가 사유가 있다면 명확히 기술

---

### 반드시 검증할 항목
- 유저 시나리오 전체 흐름 (통합 테스트)
- 로딩 / 성공 / 실패 상태에서의 유저 경험
- API 실패 시 에러 피드백
- 사용자 입력에 따른 결과 변화
- 권한 없음 / 접근 불가 상태 처리
- 순수 유틸 함수의 정확성 (단위 테스트)

### 금지 사항
- ❌ 구현 세부사항 결합 테스트 (DOM 구조, CSS 클래스, 내부 state 직접 검증)
- ❌ 단순 렌더링 확인만 하는 smoke 테스트
- ❌ 함수 호출 여부만 검증하는 spy 테스트
- ❌ "테스트 자체를 생략" — 테스트는 하되 적절한 레벨(통합)에서 한다
